Documentación de la API REST

API para gestionar empleados y proyectos en CrudMixto

Base URL: http://localhost:8080/api

Empleados API

GET /api/empleados

Obtiene todos los empleados

Respuesta:
[ { "id": 1, "nombre": "Juan Pérez", "cargo": "Desarrollador", "email": "juan@example.com" } ]
GET /api/empleados/{id}

Obtiene un empleado específico por ID

Parámetros:
  • id (Long): ID del empleado
POST /api/empleados

Crea un nuevo empleado

Body (JSON):
{ "nombre": "María García", "cargo": "Analista", "email": "maria@example.com" }
PUT /api/empleados/{id}

Actualiza un empleado existente

Parámetros:
  • id (Long): ID del empleado
Body (JSON):
{ "nombre": "María García López", "cargo": "Senior Analista", "email": "maria@example.com" }
DELETE /api/empleados/{id}

Elimina un empleado

Parámetros:
  • id (Long): ID del empleado
GET /api/empleados/search?q={query}

Busca empleados por nombre o cargo

Parámetros de consulta:
  • q (String): Término de búsqueda

Proyectos API

GET /api/proyectos

Obtiene todos los proyectos

Respuesta:
[ { "id": "507f1f77bcf86cd799439011", "nombre": "Sistema de Gestión", "descripcion": "Desarrollo de sistema interno", "empleadoId": 1, "tareasIds": ["507f1f77bcf86cd799439012"] } ]
GET /api/proyectos/{id}

Obtiene un proyecto específico por ID

Parámetros:
  • id (String): ID del proyecto (MongoDB ObjectId)
POST /api/proyectos

Crea un nuevo proyecto

Body (JSON):
{ "nombre": "Nuevo Proyecto", "descripcion": "Descripción del proyecto", "empleadoId": 1 }
PUT /api/proyectos/{id}

Actualiza un proyecto existente

Parámetros:
  • id (String): ID del proyecto
Body (JSON):
{ "nombre": "Proyecto Actualizado", "descripcion": "Descripción actualizada", "empleadoId": 2 }
DELETE /api/proyectos/{id}

Elimina un proyecto

Parámetros:
  • id (String): ID del proyecto
GET /api/proyectos/empleado/{empleadoId}

Obtiene proyectos asignados a un empleado

Parámetros:
  • empleadoId (Long): ID del empleado

Ejemplos de Uso con cURL

Crear un empleado:
curl -X POST http://localhost:8080/api/empleados \ -H "Content-Type: application/json" \ -d '{ "nombre": "Carlos Rodríguez", "cargo": "Diseñador", "email": "carlos@example.com" }'
Obtener todos los empleados:
curl -X GET http://localhost:8080/api/empleados
Buscar empleados:
curl -X GET "http://localhost:8080/api/empleados/search?q=desarrollador"
Crear un proyecto:
curl -X POST http://localhost:8080/api/proyectos \ -H "Content-Type: application/json" \ -d '{ "nombre": "Aplicación Web", "descripcion": "Desarrollo de aplicación web moderna", "empleadoId": 1 }'

Códigos de Estado HTTP

  • 200 OK: Solicitud exitosa
  • 201 Created: Recurso creado exitosamente
  • 204 No Content: Operación exitosa sin contenido de respuesta
  • 400 Bad Request: Datos inválidos en la solicitud
  • 404 Not Found: Recurso no encontrado
  • 500 Internal Server Error: Error interno del servidor
